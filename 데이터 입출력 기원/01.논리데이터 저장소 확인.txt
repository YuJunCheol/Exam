1 논리 데이터 모델 검증 ***
(1) 논리 데이터 모델 개념 : 데이터베이스 설계 프로세스의 기초적인 설계 방법이며 비즈니스 정보의 구조와 규칙을 명확하게 표현할수 있는 기법이다., 개념모델로부터 업무 영역의 업무 데이터 및 규칙을 구체적으로 표현한 모델이다.

(2) 논리 데이터 모델 특징 : 정형화, 포용성, 완전성, 독립성.

(3) 논리 데이터 모델 속성 : 개체(Entity), 속성(Attributes), 관계(Relationship)

(4) 개체-관계 (E-R) 모델
개념 : 현실 세계에 존재하는 데이터와 그들간의 관계를 사람이 이해할 수 있는 형태로 명확하게 표현하기위해 가장 널리 사용되는 모델, 논리 데이터 모델링에서는 모든 이해당사자와 의사소통의 보조 자료로 E-R 모델을 활용한다. 요구사하으로부터 얻어낸 정보들을 개체, 속성, 관계 로 기술한 모델이다.
개체-관계 다이어그램 기호 : 개체 : 사각형 관계 마름모 속성 타원 다중값 속성 이중타원, 관계-속성 연결, 선

(5) 정규화
개념 : 관계형 데이터베이스의 설계에서 중복을 최소화하여 데이터를 구조화하는 프로세스이다.

이상현상 : 데이터의 중복으로 인해 릴레이션을 조작할때 발생하는 비합리적인 현상
